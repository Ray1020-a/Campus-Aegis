# Campus Aegis 校園神盾 — 部署步驟

## 系統架構概覽

```
VyOS 路由器
  │  NetFlow UDP (port 2055)
  ▼
Campus Aegis (Node.js)
  ├── NetFlow 收集器  ─→  FlowAggregator  ─→  Gemini AI 分析
  │                                              │
  │                                    建議封鎖 IP 清單
  │                                              │
  ├── VyOS Client  ◄──────────────── 下達黑洞路由
  ├── 管理員面板 (port 3000)
  └── 誘騙服務 (port 3099)  ◄── Nginx 導向攻擊者流量
```

---

## 1. 前置需求

### 系統環境
- **作業系統**：Linux（建議 Ubuntu 22.04 / Debian 12）
- **Node.js**：v20 以上（`node --version` 確認）
- **npm**：v9 以上
- **Git**：已安裝

### VyOS 路由器需求
- VyOS 版本：1.3（Equuleus）或以上
- 已啟用 HTTPS HTTP API（見第 5 節）
- Campus Aegis 主機可透過 HTTPS 存取 VyOS 管理介面

### 網路需求
| 服務 | 協定 | 埠 | 方向 |
|------|------|----|------|
| NetFlow 收集 | UDP | 2055 | VyOS → Campus Aegis |
| VyOS API | HTTPS | 443 | Campus Aegis → VyOS |
| 管理員面板 | TCP | 3000 | 瀏覽器 → Campus Aegis |
| 誘騙服務（內部） | TCP | 3099 | Nginx → Campus Aegis |

## 4. 設定環境變數

### 4.1 建立 .env

```bash
cp .env.example .env
nano .env   # 或使用任何編輯器
```

### 4.2 必填欄位說明

| 變數 | 說明 | 範例 |
|------|------|------|
| `VYOS_URL` | VyOS HTTPS 管理介面位址 | `https://192.168.1.1` |
| `VYOS_API_KEY` | VyOS HTTP API 金鑰 | `my-secret-key` |
| `GEMINI_API_KEY` | Google Gemini API 金鑰 | `AIza...` |
| `ADMIN_PASSWORD` | 管理員面板登入密碼 | 至少 16 字元強密碼 |

### 4.3 完整環境變數參考

```ini
# VyOS 路由器（必填）
VYOS_URL=https://192.168.1.1
VYOS_API_KEY=
VYOS_INSECURE=0                   # 自簽憑證環境設為 1

# Gemini AI（必填）
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.0-flash
DEFENSE_INTERVAL_SECONDS=120      # AI 分析週期（秒）

# 管理員面板（必填）
ADMIN_PASSWORD=
ADMIN_SESSION_SECRET=             # 建議獨立隨機字串
ADMIN_PORT=3000
ADMIN_BIND=0.0.0.0

# NetFlow 收集器
NETFLOW_ENABLED=1
NETFLOW_PORT=2055
NETFLOW_BIND=0.0.0.0

# PCAP（一般不需開啟）
PCAP_ENABLED=0
PCAP_INTERFACE=eth0

# 誘騙服務
DECEPTION_PORT=3099
DECEPTION_BIND=127.0.0.1
DECEPTION_DELAY_MS=0

# 資料庫
DB_PATH=data/defense.db

# 日誌
LOG_LEVEL=info
```

## 5. VyOS 端設定

### 5.1 啟用 HTTP API

在 VyOS 執行以下指令（替換 `<YOUR_KEY>` 為自定金鑰）：

```vyos
configure
set service https api keys id campus-aegis key <YOUR_KEY>
set service https api cors allow-origin '*'
commit
save
```

將 `<YOUR_KEY>` 填入 `.env` 的 `VYOS_API_KEY`。

### 5.2 設定 NetFlow 匯出（sFlow / NetFlow v9）

```vyos
configure
set system flow-accounting interface eth0
set system flow-accounting netflow server <CAMPUS_AEGIS_IP> port 2055
set system flow-accounting netflow version 9
set system flow-accounting netflow timeout expiry-interval 60
commit
save
```

將 `eth0` 替換為 VyOS 對外的介面名稱，`<CAMPUS_AEGIS_IP>` 替換為 Campus Aegis 主機 IP。

### 5.3 確認 API 連線（從 Campus Aegis 主機測試）

```bash
curl -k -X POST https://192.168.1.1/retrieve \
  -d 'key=<YOUR_KEY>&data={"op":"showConfig","path":["protocols","static"]}'
```

回應中有 `"success":true` 即表示 API 正常。

## 6. 啟動服務

### 6.1 直接啟動（測試用）

```bash
npm start
```

啟動成功後會看到：
```
[admin] 管理員面板 http://0.0.0.0:3000
[誘騙服務] 假系統監聽 http://127.0.0.1:3099
[NetFlow] 監聽 0.0.0.0:2055
```

開啟瀏覽器前往 `http://<SERVER_IP>:3000`，輸入 `ADMIN_PASSWORD` 登入。

### 6.2 使用 PM2 守護程序（正式環境推薦）

安裝 PM2：
```bash
npm install -g pm2
```

使用內建的 PM2 設定檔啟動：
```bash
pm2 start ecosystem.config.cjs --env production
pm2 save
pm2 startup   # 依指示設定開機自啟
```

常用 PM2 指令：
```bash
pm2 status                    # 查看服務狀態
pm2 logs campus-aegis         # 即時查看日誌
pm2 restart campus-aegis      # 重新啟動
pm2 stop campus-aegis         # 停止服務
```

日誌預設儲存於：
- 標準輸出：`logs/out.log`
- 錯誤：`logs/err.log`

## 7. 驗證服務是否正常

### 7.1 Health Check API

```bash
curl http://localhost:3000/api/health
```

正常回應：
```json
{
  "success": true,
  "data": {
    "status": "ok",
    "db": true,
    "vyos": true,
    "netflowReceived": false,
    "uptimeSeconds": 10
  }
}
```

- `status: "ok"` — DB 與 VyOS 均正常
- `status: "degraded"` — DB 正常但 VyOS 無法連線
- `status: "error"` — DB 異常

### 7.2 等待 NetFlow 資料

VyOS 設定完 NetFlow 匯出後，約 1-2 分鐘開始收到流量。日誌中會出現：
```
[NetFlow] 已收到第一筆 flow（VyOS 連線正常）
```

### 7.3 確認 AI 防禦週期

預設每 120 秒執行一次 Gemini 分析，日誌中會出現：
```
[防禦] 本週期流量筆數: 152
[防禦] Gemini 未建議黑洞任何 IP   # 或
[VyOS] 已黑洞: 1.2.3.4/32
```

## 8. 進階設定

### 8.1 Nginx 反向代理（管理員面板 HTTPS）

安裝 Nginx 並建立設定檔 `/etc/nginx/sites-available/campus-aegis`：

```nginx
server {
    listen 443 ssl;
    server_name aegis.example.com;

    ssl_certificate     /etc/ssl/certs/aegis.crt;
    ssl_certificate_key /etc/ssl/private/aegis.key;

    # 管理員面板
    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name aegis.example.com;
    return 301 https://$host$request_uri;
}
```

使用 HTTPS 反代時，建議同步修改 `.env`：
```ini
ADMIN_BIND=127.0.0.1   # 僅本機監聽，由 Nginx 對外
```

### 8.2 Nginx 導向攻擊者流量至誘騙服務

若要將特定路徑（如 `/wp-login.php`、`/admin.php`）導向誘騙服務，在 Nginx 設定中加入：

```nginx
# 在上方 server 區塊內加入
location ~* \.(php)$ {
    proxy_pass http://127.0.0.1:3099;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

location ~ ^/(wp-login|wp-admin|phpmyadmin|admin\.php|shell|cmd) {
    proxy_pass http://127.0.0.1:3099;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

啟用設定：
```bash
ln -s /etc/nginx/sites-available/campus-aegis /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

### 8.3 增加防禦週期靈敏度

若網路流量大，可縮短 AI 分析週期：
```ini
DEFENSE_INTERVAL_SECONDS=60   # 每 60 秒分析一次
```

### 8.4 資料庫備份

SQLite 資料庫預設位於 `data/defense.db`，定期備份：
```bash
cp data/defense.db data/defense.db.$(date +%Y%m%d)
```

## 9. 常見問題排查

### VyOS API 連線失敗
- 確認 `VYOS_URL` 位址與埠正確
- 確認 VyOS 已執行 `set service https api` 並 `commit`
- 若使用自簽憑證，設定 `VYOS_INSECURE=1`
- 確認防火牆允許 Campus Aegis 主機連入 VyOS 443 埠

### 未收到 NetFlow
- 確認 VyOS `flow-accounting` 已設定並指向正確 IP 和埠
- 確認防火牆允許 UDP 2055 流量到達 Campus Aegis
- 使用 `tcpdump -i eth0 udp port 2055` 在 Campus Aegis 側確認封包到達

### Gemini 分析失敗（429 錯誤）
- 已內建自動重試機制（最多 2 次）
- 如持續發生，請至 Google AI Studio 確認配額
- 可適當增大 `DEFENSE_INTERVAL_SECONDS` 降低 API 呼叫頻率

### 管理員面板無法登入
- 確認 `.env` 中已設定 `ADMIN_PASSWORD`
- 同一 IP 登入失敗 5 次後會鎖定 15 分鐘
