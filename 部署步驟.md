# Campus Aegis 校園神盾 — 部署步驟

## 系統架構概覽

```
VyOS 路由器
  │  NetFlow UDP (port 2055)
  ▼
Campus Aegis (Node.js)
  ├── NetFlow 收集器  ─→  FlowAggregator  ─→  Gemini AI 分析
  │                                              │
  │                                    建議封鎖 IP 清單
  │                                              │
  ├── VyOS Client  ◄──────────────── 下達黑洞路由
  ├── 管理員面板 (port 3000)
  └── 誘騙服務 (port 3099)  ◄── Nginx 導向攻擊者流量
```

---

## 1. 前置需求

### 系統環境
- **作業系統**：Linux（建議 Ubuntu 22.04 / Debian 12）
- **Node.js**：v20 以上（`node --version` 確認）
- **npm**：v9 以上
- **Git**：已安裝

### VyOS 路由器需求
- VyOS 版本：1.3（Equuleus）或以上
- 已啟用 HTTPS HTTP API（見第 5 節）
- Campus Aegis 主機可透過 HTTPS 存取 VyOS 管理介面

### 網路需求
| 服務 | 協定 | 埠 | 方向 |
|------|------|----|------|
| NetFlow 收集 | UDP | 2055 | VyOS → Campus Aegis |
| VyOS API | HTTPS | 443 | Campus Aegis → VyOS |
| 管理員面板 | TCP | 3000 | 瀏覽器 → Campus Aegis |
| 誘騙服務（內部） | TCP | 3099 | Nginx → Campus Aegis |

## 3. 安裝依賴

```bash
npm install --omit=optional
```

> **注意**：`pcap` 為選用依賴，在 Apple Silicon（arm64）macOS 上因 `highwayhash` native module 缺乏 prebuilt binary 會編譯失敗。使用 `--omit=optional` 跳過，不影響主要功能（NetFlow 模式不需要 pcap）。
>
> Linux / x86_64 環境可正常安裝：
> ```bash
> # Linux x86_64，若需 pcap 功能
> sudo apt-get install libpcap-dev
> npm install
> ```
>
> 安裝後需單獨建置 SQLite native module（若 `better-sqlite3` 沒有對應平台的 prebuilt）：
> ```bash
> npm rebuild better-sqlite3
> ```

## 4. 設定環境變數

### 4.2 必填欄位說明

| 變數 | 說明 | 範例 |
|------|------|------|
| `VYOS_URL` | VyOS HTTPS 管理介面位址 | `https://192.168.1.1` |
| `VYOS_API_KEY` | VyOS HTTP API 金鑰 | `my-secret-key` |
| `GEMINI_API_KEY` | Google Gemini API 金鑰 | `AIza...` |
| `ADMIN_PASSWORD` | 管理員面板登入密碼 | 至少 16 字元強密碼 |

### 4.3 完整環境變數參考

```ini
# VyOS 路由器（必填）
VYOS_URL=https://192.168.1.1
VYOS_API_KEY=
VYOS_INSECURE=0                   # 自簽憑證環境設為 1

# Gemini AI（必填）
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.0-flash
DEFENSE_INTERVAL_SECONDS=120      # AI 分析週期（秒）

# 管理員面板（必填）
ADMIN_PASSWORD=
ADMIN_SESSION_SECRET=             # 建議獨立隨機字串
ADMIN_PORT=3000
ADMIN_BIND=0.0.0.0

# NetFlow 收集器
NETFLOW_ENABLED=1
NETFLOW_PORT=2055
NETFLOW_BIND=0.0.0.0

# PCAP（一般不需開啟）
PCAP_ENABLED=0
PCAP_INTERFACE=eth0

# 誘騙服務
DECEPTION_PORT=3099
DECEPTION_BIND=127.0.0.1
DECEPTION_DELAY_MS=0

# 資料庫
DB_PATH=data/defense.db

# 日誌
LOG_LEVEL=info
```

## 5. VyOS 端設定

### 5.1 啟用 HTTP API

在 VyOS 執行以下指令（替換 `<YOUR_KEY>` 為自定金鑰）：

```vyos
configure
set service https api keys id campus-aegis key <YOUR_KEY>
set service https api cors allow-origin '*'
commit
save
```

將 `<YOUR_KEY>` 填入 `.env` 的 `VYOS_API_KEY`。

### 5.2 設定 NetFlow 匯出（sFlow / NetFlow v9）

```vyos
configure
set system flow-accounting interface eth0
set system flow-accounting netflow server <CAMPUS_AEGIS_IP> port 2055
set system flow-accounting netflow version 9
set system flow-accounting netflow timeout expiry-interval 60
commit
save
```

將 `eth0` 替換為 VyOS 對外的介面名稱，`<CAMPUS_AEGIS_IP>` 替換為 Campus Aegis 主機 IP。

### 5.3 確認 API 連線（從 Campus Aegis 主機測試）

```bash
curl -k -X POST https://192.168.1.1/retrieve \
  -d 'key=<YOUR_KEY>&data={"op":"showConfig","path":["protocols","static"]}'
```

回應中有 `"success":true` 即表示 API 正常。


啟動成功後會看到：

安裝 PM2：
```bash
npm install -g pm2
```

使用內建的 PM2 設定檔啟動：
```bash
pm2 start ecosystem.config.cjs --env production

pm2 startup   # 依指示設定開機自啟
```

常用 PM2 指令：

```bash
pm2 restart campus-aegis      # 重新啟動
pm2 stop campus-aegis         # 停止服務
```

日誌預設儲存於：
- 標準輸出：`logs/out.log`
- 錯誤：`logs/err.log`

## 8. 進階設定

### 8.1 Nginx 反向代理（管理員面板 HTTPS）

安裝 Nginx 並建立設定檔 `/etc/nginx/sites-available/campus-aegis`：

```nginx
server {
    listen 443 ssl;
    server_name aegis.example.com;

    ssl_certificate     /etc/ssl/certs/aegis.crt;
    ssl_certificate_key /etc/ssl/private/aegis.key;

    # 管理員面板
    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name aegis.example.com;
    return 301 https://$host$request_uri;
}
```

使用 HTTPS 反代時，建議同步修改 `.env`：
```ini
ADMIN_BIND=127.0.0.1   # 僅本機監聽，由 Nginx 對外
```

### 8.2 Nginx 導向攻擊者流量至誘騙服務

誘騙服務現在是完整的「中互動蜜罐」，會主動偽裝成台北市立光華高中的校務系統。建議將以下攻擊者常探測的路徑全部導向 port 3099：

```nginx
# 在上方 server 區塊內加入
# PHP 檔案探測
location ~* \.(php)$ {
    proxy_pass http://127.0.0.1:3099;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

# 常見攻擊路徑
location ~ ^/(wp-login|wp-admin|wordpress|phpmyadmin|pma|phpMyAdmin|admin|administrator|manage|login|signin|dashboard|portal|shell|cmd|exec|phpinfo|info\.php|\.env|\.git) {
    proxy_pass http://127.0.0.1:3099;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

# 或直接將整台 80 port 導向誘騙服務（全面蜜罐模式）
# location / {
#     proxy_pass http://127.0.0.1:3099;
#     proxy_set_header X-Real-IP $remote_addr;
#     proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
# }
```

啟用設定：
```bash
ln -s /etc/nginx/sites-available/campus-aegis /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

### 8.3 增加防禦週期靈敏度

若網路流量大，可縮短 AI 分析週期：
```ini
DEFENSE_INTERVAL_SECONDS=60   # 每 60 秒分析一次
```

### 8.4 資料庫備份

SQLite 資料庫預設位於 `data/defense.db`，定期備份：
```bash
cp data/defense.db data/defense.db.$(date +%Y%m%d)
```

## 9. 常見問題排查

### VyOS API 連線失敗
- 確認 `VYOS_URL` 位址與埠正確
- 確認 VyOS 已執行 `set service https api` 並 `commit`
- 若使用自簽憑證，設定 `VYOS_INSECURE=1`
- 確認防火牆允許 Campus Aegis 主機連入 VyOS 443 埠

### 未收到 NetFlow
- 確認 VyOS `flow-accounting` 已設定並指向正確 IP 和埠
- 確認防火牆允許 UDP 2055 流量到達 Campus Aegis
- 使用 `tcpdump -i eth0 udp port 2055` 在 Campus Aegis 側確認封包到達

### Gemini 分析失敗（429 錯誤）
- 已內建自動重試機制（最多 2 次）
- 如持續發生，請至 Google AI Studio 確認配額
- 可適當增大 `DEFENSE_INTERVAL_SECONDS` 降低 API 呼叫頻率

### 管理員面板無法登入
- 確認 `.env` 中已設定 `ADMIN_PASSWORD`
- 同一 IP 登入失敗 5 次後會鎖定 15 分鐘

## 10. 誘騙服務（蜜罐）詳細說明

### 10.1 架構概覽

誘騙服務（`src/deception-server.js`，port 3099）是一個**中互動蜜罐**，偽裝成「台北市立光華高中校務系統」，目的是讓攻擊者相信找到真實的學校資訊系統，願意花時間操作，從而收集攻擊情報。

### 10.2 新舊版本差異（v1 → v2）

| 項目 | 舊版（v1） | 新版（v2）|
|------|-----------|----------|
| **HTTP 狀態碼** | 永遠回傳 `200` | 正確回傳 `200`/`302`/`403`/`404` |
| **HTTP Headers** | 僅 `X-Powered-By: PHP/7.4.3` | `Server: Apache/2.4.41`、`Set-Cookie: PHPSESSID`、`Cache-Control`、`Date`、`ETag` |
| **路由概念** | 無（所有路徑回傳同一內容） | 15 條明確路由，各有對應的假服務 |
| **主頁面** | 裸 HTML 登入表單（無 CSS） | 光華高中校務系統完整登入頁（有 CSS、校徽 SVG、台灣教育系統風格） |
| **登入流程** | 無（表單送出無反應） | 前 2 次登入失敗並記錄帳密，第 3 次「成功」跳轉假後台 |
| **假後台** | 無 | 學務管理後台（含學生名冊、成績異動記錄、快速連結誘餌） |
| **WordPress 假頁** | 無 | `/wp-login.php`、`/wp-admin` 完整 WordPress 登入頁 |
| **phpMyAdmin 假頁** | 無 | `/phpmyadmin` 完整 phpMyAdmin 登入介面 |
| **`/etc/passwd`** | 4 行（僅 root/daemon/www-data/mysql） | 34 行完整 Ubuntu 系統帳號，含誘餌帳號 `edu-admin`、`backup-svc` |
| **Shell 指令模擬** | 3 個指令（whoami/id/ifconfig） | 15+ 個指令（ls、pwd、ps aux、netstat、env、cat config.php 等），含誘餌帳密 |
| **phpinfo 頁面** | 5 行 HTML 空殼 | 完整 phpinfo 輸出（PHP 7.4.3、系統資訊、60+ 設定值、curl/mbstring/openssl 模組） |
| **SQL Injection 回應** | 1 筆假資料 | 5 筆學生/教職員資料（含 bcrypt 格式密碼 hash） |
| **JNDI/Log4Shell 回應** | 空回應 | 偽造 LDAP 連線 log，讓攻擊者誤以為 Log4Shell 觸發成功 |
| **`robots.txt`** | 無（同其他路徑回傳登入頁） | 揭露假的敏感路徑：`/student-data/`、`/teacher/grade-export/`、`/backup/` 等 |
| **`.env` / `.git`** | 回傳登入頁（200） | 正確回傳 403 Forbidden（Apache 標準格式） |
| **帳密記錄** | 僅 body_preview 部分截取 | 新增獨立 `credential_log` 資料表，結構化記錄 username/password/嘗試次數 |
| **Cookie/Session** | 無 | HMAC 簽名 `_sess` cookie，無狀態追蹤登入嘗試次數 |
| **favicon.ico** | 無（回傳登入頁） | 真實 ICO binary 回應 |
| **API 路徑** | 無 | `/api/*`、`/v1/*`、`/v2/*` 回傳假 JSON API 回應 |

### 10.3 蜜罐路由表

| 路由 | 偽裝服務 | 說明 |
|------|---------|------|
| `GET /` `GET /login` | 光華校務系統登入頁 | 主要入口，記錄所有訪問 |
| `POST /login` | 登入驗證 | 前 2 次失敗並記錄帳密，第 3 次 302 → /portal |
| `GET /portal` | 學務管理後台 | 含假學生名冊、成績記錄、快速功能誘餌 |
| `GET /wp-login.php` | WordPress 登入 | 偽裝學校官網後台 |
| `POST /wp-login.php` | WordPress 驗證 | 記錄帳密後回傳錯誤 |
| `GET /wp-admin*` | WordPress 後台 | 統一導向 WordPress 登入頁 |
| `GET /phpmyadmin*` | phpMyAdmin 登入 | 偽裝資料庫管理介面 |
| `GET /robots.txt` | 假 robots.txt | 揭露誘餌路徑給攻擊者繼續探索 |
| `GET /favicon.ico` | 真實 ICO | 提高瀏覽器識別的真實感 |
| `GET /.env` `GET /.git*` | 403 Forbidden | Apache 標準格式拒絕 |
| `ALL /phpinfo*` | 完整 phpinfo | PHP 7.4.3 詳細設定 |
| `ALL /etc/passwd` | 假 passwd | 34 行含誘餌帳號 |
| `ALL /api/*` `/v1/*` | 假 REST API | JSON 格式回應 |
| `ALL *`（含攻擊 payload） | 依 payload 類型 | shell/SQL/PHP exec/JNDI 各自回應 |
| `ALL *`（未命中） | 404 Not Found | Apache 標準格式 |

### 10.4 資料庫記錄

誘騙服務將所有命中記錄到兩個資料表：

**`deception_log`**（所有請求）：
```bash
sqlite3 data/defense.db "SELECT method, path, ip, user_agent, body_preview, created_at FROM deception_log ORDER BY id DESC LIMIT 20;"
```

**`credential_log`**（帳密嘗試，獨立資料表）：
```bash
sqlite3 data/defense.db "SELECT ip, username, password, attempt_num, created_at FROM credential_log ORDER BY id DESC;"
```

## 11. Demo 離線示範模式

Demo 模式可在**完全不需要 VyOS 路由器、Gemini API 金鑰、NetFlow 流量**的情況下啟動完整系統，適合展示、簡報或離線環境演示。

### 11.1 啟動方式

```bash
npm run demo
```

啟動後開啟瀏覽器前往 `http://localhost:3000`，輸入密碼 `demo1234` 登入。

### 11.2 Demo 模式與正式模式的差異

| 項目 | 正式模式 | Demo 模式 |
|------|----------|-----------|
| VyOS 連線 | 真實 HTTPS API | Mock（模擬延遲後回傳成功） |
| Gemini AI 分析 | 呼叫 Google API | Mock（1~2 秒假延遲後隨機選出封鎖 IP） |
| NetFlow 流量 | 來自路由器 UDP 封包 | 每 2 秒自動注入假流量，每 30 秒觸發一波模擬攻擊 |
| 資料庫 | `data/defense.db` | 獨立的 `data/demo.db`，不影響正式資料 |
| 設定驗證 | 啟動時檢查所有必填欄位 | 跳過，無需任何 API 金鑰 |
| 黑洞新增／撤銷 | 實際下達路由指令 | 僅寫入資料庫，不連線路由器 |

### 11.3 預置示範資料

首次啟動時，系統會自動植入過去 14 天的假資料：

- **黑洞日誌**：15 筆來自不同國家的惡意 IP，原因皆為「gemini 判定」或「管理員手動封鎖」
- **誘騙日誌**：每天 10~50 筆攻擊請求記錄（`/wp-login.php`、`/phpmyadmin`、`/.env` 等常見攻擊路徑）
- **憑證捕獲日誌**：每天 3~12 筆攻擊者嘗試的帳號密碼記錄

### 11.4 AI 防禦週期展示

Demo 模式下防禦週期設為 **60 秒**（正式預設 120 秒），啟動後約 1 分鐘即可在管理面板觀察到：

1. AI 分析流量（假延遲 1~2 秒模擬思考）
2. 自動新增黑洞 IP
3. 面板統計數字即時更新

### 11.5 清除 Demo 資料重新展示

若需要清空資料從頭演示，刪除 Demo 資料庫後重啟即可：

```bash
rm data/demo.db
npm run demo
```
